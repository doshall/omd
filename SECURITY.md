# 安全政策

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 0.1.x | ✅ 当前支持 |
| < 0.1 | ❌ 不支持 |

## 报告漏洞

如果你发现了安全漏洞，请**不要**通过公开的 GitHub Issue 报告。

请通过以下方式私下联系我们：

1. **GitHub Security Advisory**：在仓库页面点击 Security → Advisories → New draft security advisory
2. **邮件**：联系仓库维护者（见 GitHub 个人资料）

请在报告中包含：

- 漏洞描述与影响范围
- 复现步骤
- 受影响版本
- 可能的修复建议（如有）

## 响应时间

| 阶段 | 目标时间 |
|------|----------|
| 确认收到 | 48 小时内 |
| 初步评估 | 7 天内 |
| 修复发布 | 视严重程度，14–30 天内 |

## 安全最佳实践

使用 omd 时请注意：

- **Web 版**：不要编辑含敏感信息的文档；公共电脑使用后清除浏览器数据
- **桌面版**：谨慎打开来源不明的 Markdown 文件中的链接
- **部署**：自托管 Web 版时配置 CSP 头，考虑 Mermaid.js 离线部署

详细安全说明见 [docs/security.md](docs/security.md)。

# 路线图

本文档记录 omd 的功能现状与未来规划。当前 v0.1.0 **核心编辑能力已比较完整**，没有「必须马上加」的功能；以下为可选扩展方向。

> **当前用户优先级**：**A — 编辑体验**（含滚动条 Minimap）。详见 [优先实现顺序（A 类）](#优先实现顺序a-类--编辑体验)。

## 功能成熟度

作为轻量 Markdown 编辑器，omd 已具备：

- 三版本统一维护（桌面 / Web / Android）
- 实时预览、工具栏、主题、Mermaid、图片
- 桌面文件管理 / Web 自动保存

若只选几个**性价比最高**的增强，通用推荐：

1. 查找 / 替换
2. 预览区代码块语法高亮
3. 导出 HTML
4. Web PWA
5. 多标签页

#### 滚动条 Minimap（规划说明）

在编辑区（及可选预览区）滚动条一侧显示文档缩略预览，类似 IDE 中的 Minimap：

- 用色块/线条示意标题、代码块、图片等区域
- 点击或拖拽 Minimap 快速跳转
- 当前视口以高亮框标示
- **桌面版**：egui 自定义绘制于 `ScrollArea` 旁
- **Web / Android 版**：CSS + Canvas 或轻量 DOM 叠加于 `textarea` 滚动容器

## 优先实现顺序（A 类 — 编辑体验）

根据当前规划，**A 类**按推荐实现顺序排列（先易后难、先高频后低频）：

| 顺序 | 功能 | 版本 | 难度 | 说明 |
|:----:|------|------|------|------|
| 1 | 查找 / 替换 | 三版本 | 中 | ✅ 已实现 |
| 2 | 预览区代码块语法高亮 | 三版本 | 中 | ✅ 已实现 |
| 3 | 行号 + 当前行高亮 | 三版本 | 中 | ✅ 已实现 |
| 4 | **滚动条 Minimap** | 三版本 | 中高 | ✅ 已实现 |
| 5 | 编辑区与预览区同步滚动 | 三版本 | 中 | ✅ 已实现 |
| 6 | 编辑区语法高亮 | 三版本 | 高 | 需编辑器组件升级或自绘 |
| 7 | 全屏 / 专注写作模式 | 三版本 | 低 | ✅ 已实现 |
| 8 | 字号、行距、字体调节 | 三版本 | 低 | ✅ 已实现 |
| 9 | 撤销 / 重做状态提示 | 三版本 | 低 | ✅ 已实现 |
| 10 | Vim / Emacs 键位 | 桌面 / Web | 高 | 可选模式，受众较小 |

### v0.2.0 目标（A 类为主）

计划以编辑体验为主题，首批交付：

- [x] 滚动条 Minimap（桌面 / Web / Android）

## 不考虑的功能

以下不在规划范围内，以保持 omd 的轻量定位：

- 所见即所得（WYSIWYG）编辑模式
- 内置完整文件管理器 / 重量级项目树
- 博客发布 / CMS 功能
- 数据库集成
- 重量级插件市场
